# CLAUDE.md — Open School Syllabus (OSS)

## What This Project Is

OSS is a **pure data repository** — no runtime code, no server, no application logic. It contains structured, machine-readable educational curricula in YAML + Markdown, validated by JSON Schemas, enforced by CI. Licensed **CC BY-SA 4.0**.

Think "Wikipedia for curricula" — open, structured, AI-ready.

**What it is NOT:** Not a web application, API server, or learning platform. Not a place for copyrighted exam content.

**Current state:** Scaffolding phase. Documentation exists (`README.md`, `docs/`). Curriculum content, schemas, scripts, and CI workflows are being built per the [development timeline](docs/development-timeline.md).

## Tech Stack

| Technology | Purpose |
|---|---|
| YAML | All curriculum data files |
| Markdown | Teaching notes (`*.teaching.md`) |
| JSON Schema (Draft 2020-12) | Validation rules (`schema/*.schema.json`) |
| ajv-cli ≥5 | JSON Schema validator (CI + local) |
| yamllint ≥1.35 | YAML syntax and style linter |
| Node.js 20 LTS | Runtime for ajv-cli (CI only) |
| Python 3.12 | Custom validation scripts |
| GitHub Actions | CI/CD platform |

## Repository Structure

```
oss/
├── curricula/{board}/{level}/{subject-code}/
│   ├── syllabus.yaml
│   ├── subjects/*.yaml
│   ├── topics/{subject}/
│   │   ├── NN-topic-name.yaml
│   │   ├── NN-topic-name.teaching.md
│   │   ├── NN-topic-name.examples.yaml
│   │   └── NN-topic-name.assessments.yaml
│   └── locales/{lang-code}/topics/...
├── concepts/{domain}/*.yaml
├── taxonomy/{domain}/*.yaml
├── schema/*.schema.json              # 7 schema files
├── scripts/                           # Validation & maintenance scripts
├── .github/workflows/                 # CI pipelines
└── .yamllint.yml
```

## Naming Conventions

| File Type | Pattern | Example |
|---|---|---|
| Curriculum dir | `curricula/{board}/{level}/{subject-code}/` | `curricula/cambridge/igcse/mathematics-0580/` |
| Topic | `NN-kebab-case.yaml` | `05-quadratic-equations.yaml` |
| Teaching notes | same base + `.teaching.md` | `05-quadratic-equations.teaching.md` |
| Examples | same base + `.examples.yaml` | `05-quadratic-equations.examples.yaml` |
| Assessments | same base + `.assessments.yaml` | `05-quadratic-equations.assessments.yaml` |
| Translations | `locales/{ISO 639-1}/` mirrors source | `locales/ms/topics/algebra/01-expressions.yaml` |
| Concepts | `concepts/{domain}/{name}.yaml` | `concepts/mathematics/quadratic-equation.yaml` |
| Taxonomy | `taxonomy/{domain}/{area}.yaml` | `taxonomy/mathematics/algebra.yaml` |

## Schema Types (7 total)

| Schema | File | Validates | Purpose |
|---|---|---|---|
| Syllabus | `syllabus.schema.json` | `curricula/**/syllabus.yaml` | Top-level qualification |
| Subject | `subject.schema.json` | `curricula/**/subjects/*.yaml` | Groups related topics |
| Topic | `topic.schema.json` | `curricula/**/topics/**/*.yaml` (excl. examples/assessments) | Core teachable unit |
| Examples | `examples.schema.json` | `**/*.examples.yaml` | Worked examples with steps |
| Assessments | `assessments.schema.json` | `**/*.assessments.yaml` | Questions, rubrics, hints |
| Concept | `concept.schema.json` | `concepts/**/*.yaml` | Cross-curriculum bridge |
| Taxonomy | `taxonomy.schema.json` | `taxonomy/**/*.yaml` | Subject area hierarchy |

**Hierarchy:**
```
Syllabus → Subject → Topic → {Teaching Notes, Examples, Assessments}
Concept → links Topics across different Syllabi
Taxonomy → classifies Subjects into domain trees
```

## Quality Levels

| Level | Name | What's Required | AI Can Teach? |
|---|---|---|---|
| 0 | Stub | id, name, learning_objectives | Barely |
| 1 | Basic | + prerequisites, difficulty, tier, bloom_levels | Minimally |
| 2 | Structured | + teaching.sequence, common_misconceptions, engagement_hooks | Somewhat |
| **3** | **Teachable** | **+ ai_teaching_notes, examples_file, assessments_file** | **Yes — AI-ready threshold** |
| 4 | Complete | + translations (≥1 lang), cross_curriculum links, peer-reviewed | Well |
| 5 | Gold | + curriculum authority validation, student data backing | Excellently |

Level 3 is the minimum viable quality for AI tutoring.

## Provenance Tracking

| Value | Meaning | Review |
|---|---|---|
| `human` | Written by a human educator | Standard PR review |
| `ai-assisted` | Human authored with AI help | Standard PR review |
| `ai-generated` | Fully generated by AI | Educator review required |
| `ai-observed` | From student interaction data | Educator review required |

## Validation Commands

```bash
# Lint all YAML
yamllint -c .yamllint.yml curricula/ concepts/ taxonomy/

# Validate a single topic
ajv validate -s schema/topic.schema.json -d "path/to/topic.yaml"

# Validate all topics
find curricula -path "*/topics/*" -name "*.yaml" \
  ! -name "*.examples.yaml" ! -name "*.assessments.yaml" \
  | xargs -I{} ajv validate -s schema/topic.schema.json -d {}

# Custom checks
python scripts/check-prerequisites.py    # Detect prerequisite cycles
python scripts/check-references.py       # Verify cross-references
python scripts/assess-quality.py --report # Quality level report
```

## Translation Architecture

- Translations live in `locales/{lang-code}/` inside each curriculum directory
- Language codes: ISO 639-1 (e.g., `ms`, `ar`, `zh`, `hi`, `es`)
- Translation files contain ONLY translatable fields: `name`, `learning_objectives[].text`, `teaching.sequence`, `teaching.common_misconceptions`, `engagement_hooks`
- Non-translatable fields (`id`, `difficulty`, `prerequisites`, `mastery`) inherit from source

## Important Cautions

- **Copyright:** NEVER copy from copyrighted exam papers or proprietary materials. Generate original content aligned to publicly available syllabus specifications only.
- **Pedagogical accuracy:** Teaching notes, misconceptions, and sequences must be pedagogically sound. When unsure, mark provenance as `ai-generated` for educator review.
- **Provenance honesty:** Always set correct provenance. AI-written content must be `ai-generated` or `ai-assisted`, never `human`.
- **Schema compliance:** All YAML must validate against its JSON Schema. CI blocks invalid files.
- **Quality levels:** Never claim a higher `quality_level` than the content achieves. CI auto-assesses this.
- **Naming:** Follow exact naming patterns. Topic files MUST use `NN-kebab-case` format.
